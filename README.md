# 数据合规审查项目最新更新

## 2026-04-14 核心法律文本入库

- **新增正式法律文本**：已将《中华人民共和国个人信息保护法》《中华人民共和国数据安全法》《中华人民共和国网络安全法（2025修正）》三份正式法律 PDF 加入项目法规库源文件。
- **本地法规库更新**：`local-regulations.sqlite3` 已增量入库三份法律文本，当前法规库总量从 `430` 条更新为 `433` 条，其中 `national_law` 为 `3` 条。
- **分类层级修正**：导入脚本已支持将“国家法律”目录识别为 `national_law / 法律`，避免把正式法律误归入普通政策或规范文件。
- **Markdown 检索文本**：三份法律已同步导出到法规库 Markdown 检索目录，供后续检索、审查依据增强和报告生成使用。

## 2026-04-14 独立桌面安装包与 Release 分发

- **真正可分发桌面版**：macOS 安装包已改为自包含结构，应用内部直接打包 Web 资源、审查脚本、法规库与 Python 依赖，不再依赖本地项目目录才能运行。
- **双架构支持**：现已支持分别构建 Apple Silicon（`arm64`）与 Intel（`x86_64`）两个原生 `.app` / `.dmg` 安装包。
- **GitHub Releases**：安装包会统一上传到 GitHub Releases 页面，后续分发优先以 Release 附件为准。
- **文件选择器修复**：桌面壳已接入系统文件选择面板，点击“选择文件”会弹出原生 macOS 选择窗口。

## 2026-04-13 桌面版封装更新（23:53）

- **原生 macOS 应用壳**：新增 `ComplianceAI.app` 封装能力，应用以前台原生进程启动，后台拉起本地审查服务，不再依赖单独手动开浏览器。
- **DMG 分发包**：已支持生成 `ComplianceAI.dmg`，镜像中包含 `ComplianceAI.app` 与 `/Applications` 快捷方式，方便拖拽安装与对外分发。
- **文件选择器修复**：桌面壳已接入系统文件选择面板，点击“选择文件”会正常弹出 macOS 文件选择窗口，不再无响应。
- **桌面运行兼容**：Web 审查流水线的子进程统一跟随当前应用自己的 Python 解释器，避免桌面版调用系统 Python 导致环境漂移。

## 更新约定

- 从现在开始，**项目更新说明统一直接写在根目录 `README.md`**。
- `CHANGELOG.md` 保留历史记录，不再作为后续日常更新的主入口。

## 2026-04-13 二次更新（17:12）｜接入 DeepSeek API 生成专业化建议与改写条款

- **AI 驱动建议升级**：接入 DeepSeek 大模型 API，针对每条风险点生成非模板化的专业优化建议，语言更自然、贴合具体上下文，直接替代原有的规则拼接建议。
- **改写后条款支持**：在"修改建议"之外，新增"改写后条款"模块，由模型基于原文直接生成可直接替换使用的合规条款文本，不再是泛泛的示例参考。
- **高可用兜底机制**：当未配置 API Key 或调用失败时，系统自动回退到规则模板建议，确保审查流程不中断、结果始终可出。
- **前端展示优化**：风险卡片右侧单列栈式展示"修改建议"（紫）+"改写后条款"（绿），与左侧"原文摘录"+"风险分析"（柠檬黄）形成清晰的信息动线。

## 2026-04-13 前端与体验优化

- **结果页设计系统统一**：全面升级输出页视觉层级，采用 Plus Jakarta Sans + Inter/Noto Sans SC 字体体系，与首页风格保持一致。
- **风险卡片重构为左右分栏**：
  - 左侧：原文摘录 + 风险分析（新增柠檬黄浅色框，视觉层级更清晰）
  - 右侧：修改建议 + 改写后条款（由 DeepSeek 基于原文直接改写，可直接替换使用）
- **AI 增强精度优化**：接入 DeepSeek API 生成非模板化的优化建议；改写后条款基于原文直接改写，不再是通用示例。
- **字号与排版优化**：放大"审查结论""优先行动""风险分析""修改建议"等核心标签字号，强化视觉层级；增加卡片内边距与列间距，阅读更贴近专业法务报告。
- **上传页交互优化**：上传 Word/PDF 文件后，主界面文档名称自动填入文件名，减少重复输入；修复进度条消息重复堆叠的问题。
- **依赖补充**：修复因缺失 `pypdf` 导致 PDF 上传报错的依赖问题；新增 `openai>=1.30.0` 以支持 DeepSeek SDK。

## 当前已知不足

- 主法规依据目前可以稳定显示“条号 + 条款要点”，但还没有对所有核心法律实现逐条原文精确挂载。
- 结果页的“问题位置”已能提取分页与段落上下文，但默认已弱化展示，避免干扰阅读。
- 规则引擎目前仍以规则判断为主，法规库增强用于补强依据，不代表已经完成条文级法律推理。

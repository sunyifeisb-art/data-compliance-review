# 数据合规审查项目最新更新

## 2026-04-13 前端与体验优化

- **结果页设计系统统一**：全面升级输出页视觉层级，采用 Plus Jakarta Sans + Inter/Noto Sans SC 字体体系，与首页风格保持一致。
- **风险卡片重构为左右分栏**：
  - 左侧：原文摘录 + 证据片段
  - 右侧：风险分析 + 修改建议
- **字号与排版优化**：放大"审查结论""优先行动""风险分析""修改建议"等核心标签字号，强化视觉层级；增加卡片内边距与列间距，阅读更贴近专业法务报告。
- **上传页交互优化**：上传 Word/PDF 文件后，主界面文档名称自动填入文件名，减少重复输入。
- **依赖补充**：修复因缺失 `pypdf` 导致 PDF 上传报错的依赖问题。

## 2026-04-11 最新版本内容

- 完成本地法规知识库建设：已接入 `430` 份法规/标准/政策，当前全部可检索，无失败件残留。

- 报告增强升级为“条款/分块级法规支撑”：不再只做粗粒度文档命中。
- Web 结果页已接入：
  - 整改任务
  - 证据清单
  - SDK / 数据出境 / 隐私整改专项包
  - 法规库命中展示
- 上传入口已取消“文档名称必填”。
- 输入预处理已支持：
  - PDF
  - DOCX
  - DOC
- 报告下载已支持：
  - JSON
  - Markdown
- 风险展示已重构为更接近法务阅读方式：
  - 优先显示问题内容
  - 说明具体风险
  - 展示主法规依据与条款要点
  - 下方补充规范索引与证据片段

## 当前已知不足

- 主法规依据目前可以稳定显示“条号 + 条款要点”，但还没有对所有核心法律实现逐条原文精确挂载。
- 结果页的“问题位置”已能提取分页与段落上下文，但默认已弱化展示，避免干扰阅读。
- 规则引擎目前仍以规则判断为主，法规库增强用于补强依据，不代表已经完成条文级法律推理。

## 当前建议的下一步

- 给《个人信息保护法》《数据安全法》《网络安全法》等核心法律建立法条字典，做到主法规依据下直接展示条文原文或规范化条款内容。
- 继续压缩“系统判断详情”的噪音，让结果更接近“哪一页、哪段、有什么风险、依据哪条法”。
- 做一次完整端到端回归：上传 PDF → 审查 → 查看结果 → 下载 JSON/Markdown/整改清单。

# 数据合规审查项目最新更新

## 2026-04-29 数据脱敏、OCR 与法规库同步

- **新增独立产品线：数据脱敏处理**：首页现在区分「合规审查」与「数据脱敏处理」。脱敏处理不参与合规风险审查，也不生成整改任务，专门用于对用户收集到的数据做敏感信息识别、打码输出和处理报告。
- **脱敏策略升级为保留格式打码**：默认不再把手机号、邮箱、身份证、银行卡等简单替换成纯标签，而是输出更接近真实数据结构的掩码结果，例如 `138****5678`、`te***@example.com`、`110***********4512`。
- **文件格式覆盖扩展**：脱敏入口支持文本、文档、表格、JSON、图片等多类输入；页面文案改为强调「支持多种数据格式」，避免给用户造成只支持少数格式的误解。
- **OCR 能力补齐**：审查预处理与脱敏流程新增图片/PDF OCR 入口，文本型 PDF 继续走文本提取，扫描件和图片在本机具备 Tesseract 时可进入 OCR 识别链路；缺失 OCR 环境时会给出明确错误。
- **法规库重整与扩充**：本地法规 Markdown 目录重分为国家法律法规、地方规范性文件等更清晰分类，并新增境外法规资料、全文来源与构建脚本，方便后续跨境/多法域审查。
- **浏览器扩展同步**：新增 `extension/` 浏览器扩展源码、测试与构建产物，内置法规索引，可作为合规审查产品的另一种交付形态。
- **Skill 包同步**：新增 `skills/data-compliance-reviewer/`，把审查工作流、法规资料与运行脚本整理为可复用 skill 形态。
- **文档契约补充**：新增 `DESENSITIZATION_CONTRACT.md`，明确脱敏处理的接口、输入类型、输出文件、报告结构与错误码边界。

本轮验证已完成：Python 编译、脱敏策略回归、原审查 smoke check、法规库搜索、浏览器扩展测试、扩展 build、`git diff --check` 均通过。

## 2026-04-14 核心法律文本入库

- **新增正式法律文本**：已将《中华人民共和国个人信息保护法》《中华人民共和国数据安全法》《中华人民共和国网络安全法（2025修正）》三份正式法律 PDF 加入项目法规库源文件。
- **本地法规库更新**：`local-regulations.sqlite3` 已增量入库三份法律文本，当前法规库总量从 `430` 条更新为 `433` 条，其中 `national_law` 为 `3` 条。
- **分类层级修正**：导入脚本已支持将“国家法律”目录识别为 `national_law / 法律`，避免把正式法律误归入普通政策或规范文件。
- **Markdown 检索文本**：三份法律已同步导出到法规库 Markdown 检索目录，供后续检索、审查依据增强和报告生成使用。

## 2026-04-13 二次更新（17:12）｜接入 DeepSeek API 生成专业化建议与改写条款

- **AI 驱动建议升级**：接入 DeepSeek 大模型 API，针对每条风险点生成非模板化的专业优化建议，语言更自然、贴合具体上下文，直接替代原有的规则拼接建议。
- **改写后条款支持**：在"修改建议"之外，新增"改写后条款"模块，由模型基于原文直接生成可直接替换使用的合规条款文本，不再是泛泛的示例参考。
- **高可用兜底机制**：当未配置 API Key 或调用失败时，系统自动回退到规则模板建议，确保审查流程不中断、结果始终可出。
- **前端展示优化**：风险卡片右侧单列栈式展示"修改建议"（紫）+"改写后条款"（绿），与左侧"原文摘录"+"风险分析"（柠檬黄）形成清晰的信息动线。

## 当前已知不足

- 主法规依据目前可以稳定显示“条号 + 条款要点”，但还没有对所有核心法律实现逐条原文精确挂载。
- 结果页的“问题位置”已能提取分页与段落上下文，但默认已弱化展示，避免干扰阅读。
- 规则引擎目前仍以规则判断为主，法规库增强用于补强依据，不代表已经完成条文级法律推理。
